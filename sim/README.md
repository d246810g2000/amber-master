# 🏸 排點演算法模擬分析工具 (Amber Badminton Sim)

本目錄包含本球會「排點引擎」的壓力測試與量化分析工具。我們使用模擬器來確保球友在現場能獲得最佳的「新鮮感」與「對戰流暢感」。

## 🚀 核心腳本

- **[advancedSim.ts](./advancedSim.ts)**: 最全面性的分析腳本。
  - 模擬場景：7:00~9:00 PM (120分鐘)，16 人規模，2 面場地。
  - 特色：模擬球員陸續進場、蒙地卡羅 (Monte Carlo) 多次迭代分析。
- **[SimulationRunner.ts](./SimulationRunner.ts)**: 模擬器底層引擎。
  - 負責處理時間步進、戰力 (TrueSkill) 更新、比賽生命週期管理。

## 📊 三大監測指標 (Key Metrics)

好的排點演算法必須在以下三個指標中取得平衡：

1. **新鮮度 (Diversity)**
   - **目標**：絕對不重複出現相同的四人組合，且盡量不重複搭檔。
   - **達成方式**：使用「全組合掃描 (Exhaustive Scan)」技術，從 $C(n, 4)$ 種可能性中過濾掉已發生的組合。

2. **公平性 (Fairness)**
   - **目標**：確保每位球友整晚的總場次落差在 1 場以內。
   - **達成方式**：優先挑選 `matchCount` 較少的人進場。

3. **流暢感 (Flow / Waiting Experience)**
   - **目標**：**不連休兩場**。即便有人遲到，也不要讓早到的人在場下枯等太久。
   - **達成方式**：引入「等候權重 (Wait count bonus)」。連休兩場後會獲得強制優先權。

## 🛠️ 如何執行

確保您已安裝 dependencies 之後，直接執行：

```bash
npx tsx sim/advancedSim.ts
```

## 📝 執行範例報告

```text
==================================================
📊 統計分析報告 (n = 20)
==================================================
1. 平均每晚總場次: 23.5 場
2. 組合多樣性:
   - 平均重複四人組次數: 0.5 次/週
   - 平均重複搭檔次數: 0.1 次/週
3. 等候感 (等候優化模式):
   - 平均最長連休場數: 1.8 場 (意即：打一休一，很少連休兩場)
   - 場數標準差: 0.45 (極高公平)
```
